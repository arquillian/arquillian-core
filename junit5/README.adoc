= Arquillian JUnit Jupiter Integration

This module provides Arquillian integration for JUnit Jupiter.

== JUnit Compatibility

This module supports the *JUnit Jupiter* platform.

[IMPORTANT]
====
Despite the `junit5` naming convention, this module provides first-class support for both **JUnit 5** and **JUnit 6**.
The module name reflects the integration origin rather than a version limitation.
====

== JDK Compatibility

The minimum JDK requirement depends on the version of JUnit used:

* **Using JUnit 5.x** requires **Java SE 8** or later.
* **Using JUnit 6.x** requires **Java SE 17** or later.

== Usage

[source,xml]
----
<dependency>
    <groupId>org.jboss.arquillian.junit5</groupId>
    <artifactId>arquillian-junit5-container</artifactId>
    <version>${org.arquillian.version}</version>
</dependency>
----

== Configuration

=== `arquillian.junit5.manager.perClass`

JUnit configuration parameter. Default: `false`.

The value is read via JUnit's
`ExtensionContext.getConfigurationParameter(String)`, so it can be supplied
through any of the JUnit Platform's standard channels — a JVM system property
(`-Darquillian.junit5.manager.perClass=true`), a `junit-platform.properties`
file on the test classpath, or Launcher-supplied configuration.

When `false` (the default), a single `TestRunnerAdaptor` / Arquillian `Manager`
is shared across every test class in the JUnit run. `BeforeSuite` /
`AfterSuite` fire once per JVM. This matches the classic Arquillian model and
is the correct setting for shared managed external containers (WildFly,
Payara, GlassFish, …) that boot once and run every test class against the
same instance.

When `true`, each test class gets its own `TestRunnerAdaptor` and underlying
`Manager`. `BeforeSuite` / `AfterSuite` therefore fire once per class. Enable
this for **parallel class execution**
(`junit.jupiter.execution.parallel.mode.classes.default=concurrent`) against
isolating embedded containers that allocate their own resources per class
(e.g. embedded Jetty on port `0`). A shared manager in that mode would leak
thread-local context activations between classes running on different
threads.

Enable it via `src/test/resources/junit-platform.properties`:

[source,properties]
----
arquillian.junit5.manager.perClass=true
----

Or from Maven Surefire:

[source,xml]
----
<systemPropertyVariables>
    <arquillian.junit5.manager.perClass>true</arquillian.junit5.manager.perClass>
</systemPropertyVariables>
----
